# 微信 45166 真实上传排除法设计

**目标**

在 Obsidian 插件内部增加一个“定位当前文档出错元素”的真实上传排查入口，复用现有发布链路，将最终待上传的 HTML 进行分块、真实调用微信 `draft/add` 接口、逐步缩小到最小失败子集，并把完整试投记录落到 `_mofa_debug/bisect/`。

**范围**

- 复用现有 token、图片上传、封面推断和 HTML 清洗逻辑。
- 不新增新的外部依赖，不引入新的配置项。
- 输出 JSON 报告和 HTML 片段，记录每次请求的完整入参、原始文档 ID、请求时间、响应时间和回包。

**方案**

1. 先使用当前正式发布链路生成 `sanitizedHtml`。
2. 以 `.mofa-article` 的直接子节点作为一级候选块。
3. 对候选块数组使用 delta debugging：
   - 若某个子集单独上传仍触发 `45166`，继续缩小到该子集。
   - 若某个补集触发 `45166`，继续缩小到该补集。
   - 若最小失败集合仍包含多个块，则报告“组合触发”。
4. 若最小失败集合只剩 1 个块，则尝试进入该块的子节点继续缩小。
5. 输出 `_mofa_debug/bisect/*.json` 和 `_mofa_debug/bisect/*.html`。

**错误处理**

- 若完整文档已经上传成功，则终止排查并提示“不复现”。
- 若试投过程中出现非 `45166` 错误，例如 `45009`、`40164`、`40014`，立即中止并把该响应写入报告。
- 若候选块无法继续拆分，则返回当前最小失败块。

**测试**

- 为 delta debugging 逻辑补纯函数测试。
- 为标签预览和文本裁剪补测试。
- 已有的 `sanitizeForWechat` 与调试日志测试继续保留，作为真实上传排查的底层保障。
